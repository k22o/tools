# 認証認可

## 認証方法

### パスワードによる認証プロトコル
  
- ユーザ名やPWなどを平文で送信する
  - 後述するCHAPとの対比で、PAP (Password Authentication Protocol) とも
  - telnet、Basic認証
- PWをhashなどで変換して送る
- **Chanllenge/Response認証**
  - 認証の際にサーバから乱数を送り、それをhashに含めてもらう
  - CHAP (Challenge Handshake Authentication Protocol) とも

### 公開鍵ペアを用いた認証プロトコル

- 公開鍵ペアを利用して認証する
- 公開鍵ペアとデジタル署名を使った認証

### ID連携

- 別々の組織で運営されるサービス間で、IDや付随した属性情報を連携する仕組みを**ID連携**と呼ぶ。
- ID連携を利用して、1度ログインしたら他ではログインする必要のない仕組みを、**SSO (Single Sign ON)** と呼ぶ。
- ID連携の実現には、OpenIDとSAMLの2つの方式が主流。

### webにおける認証プロトコル

- **BASIC認証**
  - ユーザ名とPWをBase64変換で送信
- **Digest認証**
  - サーバからのchallengeとPWなどをハッシュ化して送信
- cookieを利用したセッション管理

### 認証サーバ

ネットワークなどアクセスしようとしているユーザーの認証認可を担うする専用サーバを認証サーバと呼ぶ。
**RADIUS**や**LDAP**といった認証プロトコルが利用される。

### FIDO/パスキー

- **FIDO (Fast IDentity Online)**
  - パスワードに依存しない認証を実現するための標準規格。FIDO Allianceによって策定される。
  - 公開鍵暗号方式を採用し、端末内で生成した秘密鍵は端末外に出さず、公開鍵のみをサービス側に登録する。認証時は生体認証やPINなどで端末側の秘密鍵を利用して署名を行う。
  - サーバ側にパスワードやその情報が保存されないため、フィッシングやパスワード漏えいによる不正ログインに強い。
  - **FIDO2**は、ブラウザから利用するためのWeb標準API (**WebAuthn**) と、認証器との通信仕様 (**CTAP**) から構成される、現在主流のFIDO規格。
- **パスキー (Passkey)**
  - FIDO2/WebAuthnの仕組みを利用した認証情報 (鍵ペア) そのもの、またはその利用体験を指す呼び方。
  - スマートフォンやPCなどの端末、あるいはOSベンダーのクラウド (iCloudキーチェーン、Googleパスワードマネージャー等) を通じて複数端末間で同期でき、機種変更時にも再登録なしで利用しやすい。
  - パスワード不要でIDとパスキーのみでログインできるため、パスワードの使い回しや漏えいによるリスクを低減できる。
