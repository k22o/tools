# 代表的な攻撃

※ 個別には記載していないが、同一オリジンポリシーやCORSの設定も攻撃の対策に寄与している

## 脆弱性に対する攻撃

CPU・ファームウェア・アクセスに伴う脆弱性を利用した攻撃。

### Buffer Overflow

文字入力を利用した攻撃で、バッファを超えた入力を与えることで、予期せぬメモリの値を書き換える。
対策は以下の通り。

- コンパイル時にスタック保護の機能を利用する
- あらかじめ定めた上限の領域を確保して、そこまでの文字数しか受け付けなくする

### OSコマンドインジェクション

SQLインジェクションのOS版。OSのコマンドを実行させる。
対応策としては、以下の通り。

- エスケープ処理
- そもそもOSコマンドをアプリケーションからの入力をもとに実行しない

### サプライチェーン攻撃

ソフトウェアやハードウェアの開発・供給網(サプライチェーン)に侵入し、正規の製品やアップデートに不正なコードを混入させることで、最終的な利用者に被害を及ぼす攻撃。信頼された配布経路を悪用するため検知が難しい。
対策は以下の通り。

- 依存ライブラリやOSSの脆弱性を継続的にスキャンする (SCA)
- リリース後、**日以内のOSSは利用しない
- ソフトウェア部品表 (SBOM) を管理し、利用している構成要素を可視化する
- 取引先・委託先のセキュリティ体制を審査する
- コード署名やハッシュ値検証により配布物の改ざんを検知する

### ゼロデイ攻撃

脆弱性が発見されてから修正パッチが提供されるまでの間 (ゼロデイ) に、その未対応の脆弱性を突いて行われる攻撃。防御側に対策の猶予がないため被害が拡大しやすい。
対策は以下の通り。

- IDS/IPSやWAFなどにより不審な挙動を検知する
- 最小権限の原則に基づきシステムへのアクセスを制限する
- サンドボックスや仮想化により被害範囲を局所化する
- ベンダーの脆弱性情報を継続的に収集し、パッチ適用を迅速化する


## webアプリケーションの脆弱性に対する攻撃

### XSS

不正なスクリプトを攻撃対象ページに挿入して、不正スクリプトを実行させる。
対策は以下の通り。

- エスケープ -> [別ファイル](../sanitize.md)
- 入力ボックスで想定される文字 (e.g. 数字) 以外は受け付けないようにする
- DOM操作用のメソッドなどを利用
  - innerHTMLでdomを差し込まない

### CSRF

正式なログイン用cookieなどを利用して罠サイトを通して不正アクセスさせる。
対策は以下の通り。

- トークンの利用
- double submit cookieの利用
- cookieにsame site属性を設定
- originヘッダの値を都度検証する

### クリックジャッキング

ユーザの意図とは異なるボタン等をクリックさせて意図しない挙動を実行させる。例えば、iframeで埋め込んだページの上に罠サイト独自のボタンを置き、罠サイトのボタンを押したつもりがその下のiframeを実行してしまう。  
x-frame-optionsを利用して防ぐ。

### オープンリダイレクト

webアプリケーションのリダイレクト機能を利用して罠サイトなどに遷移させる。たとえばクエリパラメータでリダイレクト先を渡したり、入力値に応じてリダイレクト先を指定する場合に発生する。
リダイレクト先urlの検証コードを入れることで防止できる。

### SQLインジェクション

不正なコードを入力値として与えることで。SQL文が実行された際にDBのデータを入手したり破壊したりする。
対応策としては、以下の通り。

- prepared statementの機能を利用してエスケープする
- 入力ボックスで想定される文字 (e.g. 数字) 以外は受け付けないようにする

## パスワード系

- **ブルートフォース攻撃**
   - 主にパスワードを不正に入手するために用いられ、考えられるパターンを総当たり的に試す。
   - 対応策としては、以下の通り。
      - パスワードを複雑にさせる (桁数・文字種)
      - ワンタイムパスワードや生体認証、2FAの利用
      - 一定回数のミスでブロック
- **辞書攻撃**
   - 流出したパスワードや一般に使われる文字列をもとに攻撃する
- **レインボーテーブル攻撃**   
   - ハッシュ化されたパスワードを事前計算してDBで照合する


## なりすまし

攻撃者が正規の利用者やサービスを装って被害者を騙し、情報の詐取や不正操作を行う攻撃の総称。以下のような手法がある。

- **メールスプーフィング**
  - 送信元メールアドレスを偽装し、実在する組織や人物からのメールであるかのように見せかけて送りつける手法。
- **DNSスプーフィング**
  - DNSの応答を偽装・改ざんし、利用者を本来とは異なる不正なサーバへ誘導する手法。
- **セッションハイジャック**
  - 通信の盗聴や予測などによって正規利用者のセッションID (Cookieなど) を奪取し、そのセッションになりすまして不正操作を行う手法。

## フィッシング

実在する企業やサービスを装った偽のメールやサイトに誘導し、ID・パスワードやクレジットカード情報などを入力させて詐取する攻撃。以下のような手法がある。

- **メールフィッシング**
  - 実在する企業やサービスを装った偽のメールを不特定多数に送りつけ、偽サイトへ誘導して情報を詐取する手法。
- **スピアフィッシング**
  - 特定の個人や組織を標的に、業務内容や人間関係を調べた上で送る、より巧妙で説得力の高いフィッシングメールを用いる手法。
- **SMSフィッシング**
  - SMS (ショートメッセージ) を用いて偽サイトへ誘導するフィッシングの手法。宅配便や金融機関を装うことが多い。
- **ファーミング**
  - hostsファイルやDNSの改ざんにより、正しいURLを入力しても偽サイトへ誘導されるようにする手法。利用者が気づきにくい。
- **QRコードフィッシング**
  - 偽のQRコードを読み取らせることで偽サイトへ誘導し、情報を詐取する手法。

## ソーシャルエンジニアリング

技術的な手段ではなく、人の心理的な隙や行動のミスにつけこんで機密情報を入手したり不正な操作を行わせたりする攻撃手法。以下のような手法がある。

- **pre-texting攻撃**
  - 上司・取引先・情報システム部門などを装ったもっともらしい口実 (pretext) を作り上げ、それを信じ込ませることで機密情報を聞き出す手法。
- **Baiting攻撃**
  - USBメモリなど興味を引く「餌 (bait)」を意図的に放置・配布し、被害者に取り込ませることでマルウェアに感染させる手法。
- **ビジネスメール詐欺**
  - 経営者や取引先になりすまし、偽の請求書や送金指示のメールを送りつけて金銭を詐取する手法 (BEC)。
- **Watering Hole攻撃**
  - 標的がよく利用するWebサイトを事前に改ざんしておき、そのサイトを訪問した際にマルウェアへ感染させる手法。

## SEO系

- **SEOポイズニング**
   - 検索再度の順位アルゴリズムを悪用して、悪意のあるサイトの順位を上げること
- **SEOスパム**
   - 広告収入やアクセス数のために、キーワードと関係の薄いサイトを不正に上位に表示すること

## その他

- **サイドチャネル攻撃**
   - 非破壊攻撃の一種。攻撃対象の物理デバイスから得られる物理量 (処理時間や消費電力など) やエラーメッセージから、内部情報を解析する。
   - 暗号アルゴリズムを実装したシステムに適用することで、暗号解読の手がかりとなる。
- **中間者攻撃**
   - 通信を行う2者間に割り込んで、情報をすり替えることで、その後の通信を盗聴する。
- **テンペスト攻撃**
   - 処理中に機器から放出される電磁波を利用して、キー入力や画面表示などの情報を解析する。
- **標的型攻撃**
   - 特定の個人や組織を狙った攻撃
