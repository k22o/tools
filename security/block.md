# 脅威への対策

セキュリティでは、以下の3つの脅威に対応する。
- 技術的脅威 (マルウェア・ハッキング etc)
- 物理的脅威 (不正アクセス・盗難 etc)
- 人的的脅威 (ソーシャルエンジニアリング・人的ミス etc)

## 情報セキュリティの目標

- 情報セキュリティの3要素 (CIA)
   - **機密性 (Confidentiality)**: 許可された者だけが情報にアクセスできる状態を確保すること
   - **完全性 (Integrity)**: 情報が改ざん・破壊されず、正確かつ最新の状態に保たれていること
   - **可用性 (Availability)**: 許可された者が必要なときに情報やシステムに問題なくアクセスできること
- 情報セキュリティの7要素
   - 上記CIAの3要素に加え、以下の4要素を含めたもの
   - **真正性 (Authenticity)**: 情報やアクセスしている主体が、なりすましではなく本物であることを確実にすること
   - **責任追跡性 (Accountability)**: 誰がいつどのような操作を行ったかを追跡・証明できること
   - **否認防止 (Non-repudiation)**: 情報の発信や操作を行った本人が、後からその事実を否認できないようにすること
   - **信頼性 (Reliability)**: システムが意図した通りに一貫して矛盾なく動作すること


## 対策に関連した概念

### サイバーキルチェーン

Lockheed Martin社が提唱した、サイバー攻撃を偵察から目的の達成までの一連のプロセスとしてモデル化したもの。各段階を理解することで、どの段階で攻撃を検知・遮断すべきかを検討しやすくなる。

1. **偵察 (Reconnaissance)**: 標的の情報 (組織構成・使用システム・従業員情報など) を収集する
2. **武器化 (Weaponization)**: 収集した情報をもとにマルウェアなどの攻撃ツールを準備する
3. **配送 (Delivery)**: メールやWebサイトなどを通じて武器化したツールを標的に送り込む
4. **攻撃 (Exploitation)**: 標的システムの脆弱性を突いてツールを実行させる
5. **インストール (Installation)**: バックドアなどを設置し、標的環境への足がかりを確保する
6. **遠隔操作 (Command & Control)**: C&Cサーバと通信を確立し、外部から標的を制御可能にする
7. **目的の実行 (Actions on Objectives)**: 情報窃取・データ破壊・ランサム要求など、当初の目的を達成する

### Mitre ATT&CK

MITRE社が公開している、実際に観測された攻撃者の戦術 (Tactics) と技術 (Techniques) を体系的にまとめたナレッジベース。サイバーキルチェーンより粒度が細かく、攻撃者がどのような手口 (例: 初期アクセス・権限昇格・防御回避など) を用いるかがマトリクス形式で整理されている。
脅威モデリングや検知ルールの設計、レッドチーム演習のシナリオ作成など、防御・攻撃双方の観点で共通言語として活用される。

### ゼロトラストモデル

従来のセキュリティ対策は、ファイアーウォールなどで社内ネットワークと外部を明確に区切り、内部は安全であるという前提のもとに外部からの侵入のみを防ぐ**境界防御モデル**が主流だった。しかし、リモートワークの普及やクラウド利用の拡大により内外の境界が曖昧になったことに加え、内部の端末やアカウントが一度でも侵害された場合に内部での被害拡大を防げないという弱点があった。

**ゼロトラストモデル**は、「社内ネットワークだから安全」という暗黙の信頼を置かず、社内・社外を問わずすべての通信やアクセスを都度検証するという考え方に基づくセキュリティモデル。具体的には以下のような対策を組み合わせて実現する。

- アクセスの都度、ユーザー・デバイス・通信を認証・検証する (暗黙の信頼をしない)
- 必要最小限の権限のみを付与する (最小権限の原則)
- ネットワーク・端末・アプリケーション・データなど複数の層で対策を重ねる**多層防御**の考え方を取り入れ、単一の防御が突破されても被害を局所化する
- 通信ログを継続的に監視し、異常があれば即座に検知・対応する

境界防御モデルが「外側からの侵入を防ぐ」ことに主眼を置くのに対し、ゼロトラストモデルは「内部に侵入された前提」で被害を最小化する考え方といえる。

ゼロトラストモデルを実際に実現するにあたっては、以下のような技術・仕組みが用いられる。

- **動的アクセス制御 (Dynamic Access Control)**
  - ユーザーの属性・所属デバイスの状態・アクセス元の場所や時間帯など、状況に応じてアクセスの可否をそのつど動的に判断する仕組み。「暗黙の信頼をしない」というゼロトラストの考え方を実装レベルで支える中核要素。
- **SIEM (Security Information and Event Management)**
  - ファイアーウォールやIDS/IPS、サーバなど複数の機器・システムから出力されるログやイベント情報を一元的に収集し、相関分析することで脅威の兆候を検知する仕組み。
- **UEBA (User and Entity Behavior Analytics)**
  - ユーザーや端末などの普段の挙動を学習し、そこから逸脱する異常な振る舞い (普段と異なる時間帯・場所からのアクセスなど) を検知する技術。SIEMの検知精度を高めるために用いられることが多い。
- **XDR (Extended Detection and Response)**
  - 後述のEDRの発展形。エンドポイントだけでなく、ネットワーク・クラウド・メールなど複数レイヤーの検知情報を横断的に統合・相関分析し、脅威の検出から対応までを行う。
- **SOAR (Security Orchestration, Automation and Response)**
  - SIEMやXDRなどが検知したアラートに対する調査・対応の手順を自動化・オーケストレーションし、セキュリティ運用の効率化と対応の迅速化を図る仕組み。
- **CASB (Cloud Access Security Broker)**
  - 利用者と各種クラウドサービスとの間に立ち、企業のセキュリティポリシーの適用・可視化・制御を行う仕組み。許可されていないクラウドサービスの利用 (シャドーIT) の検知などに用いられる。
- **SWG (Secure Web Gateway)**
  - 利用者がインターネット上のWebサイトへアクセスする通信を中継・検査し、マルウェアの侵入やポリシー違反のアクセスをブロックするゲートウェイ。
- **SASE (Secure Access Service Edge)**
  - SD-WANなどのネットワーク機能と、CASBやSWG、ZTNA (ゼロトラストネットワークアクセス) などのセキュリティ機能をクラウド上で統合的に提供するアーキテクチャ。拠点や端末の場所によらず一貫したゼロトラストポリシーを適用できる。



## ネットワークでのセキュリティ

### ファイアーウォール

内部ネットワークと外部ネットワークの境界に設置し、通信を許可・拒否するルールに基づいて不正な通信を遮断する仕組み。

- **パケットフィルタリング型**
  - TCP/IPの (2) インターネット層・(3) トランスポート層で機能する。送信元/宛先IPアドレス、ポート番号、プロトコルの種類などパケットのヘッダ情報をもとに通信の可否を判定する。処理が軽い一方、通信内容までは検査しない。
- **ステートフルインスペクション型**
  - パケットフィルタリング型と同様に (2) インターネット層・(3) トランスポート層で機能するが、通信のセッション状態 (コネクションの確立状況) を管理・追跡し、一連の通信の文脈に基づいて判定する。戻りの通信のみを自動的に許可するなど、より柔軟な制御が可能。
- **アプリケーション型**
  - (4) アプリケーション層で機能する。HTTPなどアプリケーションプロトコルの内容 (URLやリクエストの中身など) まで検査できるため、より詳細な制御が可能。WAF (Web Application Firewall) はこの一種。処理負荷は他の方式より高い。

### IDS, IPS

- **IDS (Intrusion Detection System / 侵入検知システム)**
  - ネットワークやサーバ上の通信・ログを監視し、不正アクセスや攻撃の兆候を検知して管理者に通知する。検知のみを行い、通信の遮断などの防御動作は行わない。
- **IPS (Intrusion Prevention System / 侵入防止システム)**
  - IDSの検知機能に加えて、不正な通信を検知した際に自動的に遮断・ブロックするなどの防御動作までを行う。

検知方式には、既知の攻撃パターンと照合する**シグネチャ型 (パターンマッチング型)** と、通常時の挙動から外れた異常を検知する**アノマリ型**がある。


### VPN

インターネットなどの公衆網上に、暗号化によって保護された仮想的な専用線 (トンネル) を構築し、拠点間やリモート端末から社内ネットワークへ安全にアクセスできるようにする技術。

- **SSL-VPN**
  - SSL/TLSを利用してアプリケーション層 (トランスポート層以上) でトンネルを構築する。Webブラウザから利用できるものが多く、専用クライアントなしで手軽に導入しやすい。443番ポートを使うためファイアーウォール越しでも利用しやすい。
- **IPsec-VPN**
  - IPsecプロトコルを利用してインターネット層 (IPパケットそのもの) を暗号化する。拠点間 (サイト間) VPNなど、ネットワーク全体を継続的に接続する用途でよく使われる。専用クライアントやルータ側の設定が必要になることが多い。

## 端末でのセキュリティ

- **EDR (Endpoint Detection and Response)**
  - PCやサーバなどのエンドポイント上での不審なプロセス実行やファイル操作といった挙動を常時監視し、脅威の検知・調査・対応 (該当端末のネットワークからの隔離など) を行う仕組み。
  - 既知のパターンで侵入自体を防ぐことに主眼を置く従来のアンチウイルス (パターンマッチング法) とは異なり、「侵入されること」を前提に、侵入後の不審な挙動をいち早く検知し被害を最小化することを目的とする。
