# 物理層・データリンク層

## インターフェース

- **リピータ**
  - L1層で、ネットワークを延ばすためのもの
- **ブリッジ**
  - L2層でネットワーク同士をつなぐ
  - MACアドレスで処理する

## プロトコル

### **Ethernet**
  
- LAN（Local Area Network）で最も広く使われている有線ネットワーク技術で、コンピューターやネットワーク機器間でデータを送受信するためのプロトコル
- かつてはCSMA/CD方式
- 同軸上でつなぐ方式からスイッチになったことで、Collision対策が不要になり、CMSA/CDも不要に

### 無線通信

- **Wi-Fi** (IEEE 802.11)
  - 無線LANで使用されるプロトコルで、ワイヤレスネットワーク上でのデータ通信を規定。
  - WEPやWPAは、現在は推奨あれない
  - CMSA/CA方式
    - セキュリティ -> WPA2, WPA3
- BlueTooth
- ZigBee
  - 家電等への組み込み前提、低消費電力で短距離の無線通信
- LPWA
  - IoTなど、低消費電力・長距離の通信

### PPP

- **PPP (Point-to-Point Protocol)**
  - ポイントツーポイント接続で使用されるプロトコルで、シリアル通信やダイヤルアップ接続で利用
    - シリアル通信: 伝送路を1本、または2本使用して、データを1ビットずつ連続的に送受信する通信方式
- **PPPoE (Point-to-Point Protocol Ethernet)**
  - PPPをLAN上で実現する

## ループの検出

ブリッジなどの攻勢でループがあると、情報が永久に回り続けてシステムトラブルの原因になる。そこでいくつかの検出技術がある。

- **スパニングツリー**
- **リンクアグリゲーション**
- **LLDP**

## VLAN

- ネットワークの配線を変えずにネットワーク構造を変える
- ブリッジによっては組み込まれている機能
- 大きく2種類に分けられる
  - **スタティックVLAN**: あらかじめスイッチなどのポートにVLANを設定する方法
  - **ダイナミックVLAN**: 接続されるデバイスに応じてポートが動的に所属VLANを変更する


## 備考

### CSMA/CD 方式

**CSMA/CD (Carrier Sense Multiple Access with Collision Detection)** は、かつての同軸ケーブル (バス型) のEthernetで使われていた通信制御方式。

1. **キャリアセンス**: 送信前に伝送路が使用中でないか (他の信号が流れていないか) を確認する
2. **マルチプルアクセス**: 複数の端末が同じ伝送路を共有し、空いていれば誰でも送信できる
3. **コリジョン検出 (Collision Detection)**: 複数端末が同時に送信して信号が衝突 (コリジョン) した場合、それを検出し、送信を中断してジャム信号を送出したのち、ランダムな時間待機してから再送する

現在の主流であるスイッチを用いたEthernetでは、ポートごとに衝突ドメインが分離され全二重通信が可能になったため、コリジョン自体が発生せず、CSMA/CDは実質的に不要となっている。

### CSMA/CA 方式

**CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)** は、主にWi-Fiなどの無線LANで使われる通信制御方式。無線では有線のように衝突を確実に検出することが難しい (隠れ端末問題など) ため、検出ではなく事前の**回避 (Avoidance)** に重点を置く。

1. **キャリアセンス**: 送信前に伝送路が空いているかを確認する
2. **ランダムバックオフ**: 伝送路が空いていても即座に送信せず、ランダムな待機時間を挟んでから送信することで、他端末との送信タイミングの衝突を避ける
3. **ACK応答**: 受信側は正常に受信できた場合にACK (受信確認) を返す。送信側はACKが返ってこない場合、衝突や電波障害が発生したとみなし再送する
4. 必要に応じて**RTS/CTS**による事前のネゴシエーションを行い、隠れ端末問題による衝突を回避することもある
